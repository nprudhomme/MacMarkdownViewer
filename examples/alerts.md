# GitHub Alerts

GitHub-style alert blocks for highlighting important information.

## Note

> [!NOTE]
> This is a note alert. Use it to highlight information that users should take into account, even when skimming.

## Tip

> [!TIP]
> This is a tip alert. Use it to provide helpful advice for doing things better or more easily.

## Important

> [!IMPORTANT]
> This is an important alert. Use it for crucial information necessary for users to succeed.

## Warning

> [!WARNING]
> This is a warning alert. Use it when urgent attention is needed due to potential risks.

## Caution

> [!CAUTION]
> This is a caution alert. Use it to advise about negative potential consequences of an action.

## Alerts with Rich Content

> [!NOTE]
> Alerts can contain **bold**, *italic*, `code`, and [links](https://example.com).
>
> They can also contain multiple paragraphs and lists:
> - First item
> - Second item
> - Third item

> [!TIP]
> You can even include code blocks inside alerts:
>
> ```bash
> echo "Hello from inside an alert!"
> ```

> [!WARNING]
> Be careful when combining alerts with other markdown features.
> Always test your content to ensure proper rendering.
>
> | Feature | Supported |
> |---------|-----------|
> | Bold | Yes |
> | Code | Yes |
> | Lists | Yes |
